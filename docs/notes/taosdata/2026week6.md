## 各工具的扫描范围

并非所有 Agent CLI 都会扫描全部三个目录，具体取决于各家实现：

- GitHub Copilot CLI — 扫描全部三个（.github/、.claude/、.agents/），兼容性最广
- Claude Code — 主要扫描 .claude/ 和 .agents/
- 其他实现 — 只要声明支持 Agent Skills 开放规范，至少会扫描 .agents/

因此，如果希望编写一个跨工具通用的 Skill，将其放在 .agents/skills 目录下是最稳妥的选择。

## 如何安装skill

- 方式一：放入项目约定目录（项目级，仅当前项目生效）
  your-project/
  ├── .github/skills/my-skill/SKILL.md
  ├── .claude/skills/my-skill/SKILL.md # 或
  └── .agents/skills/my-skill/SKILL.md # 或
  该 Skill 只在当前项目中生效，不影响其他项目。适合与项目代码一起版本管理。

- 方式二：放入个人约定目录（个人级，所有项目生效）
  将 Skill 文件夹放到用户主目录下的约定路径中：
  ~/.copilot/skills/my-skill/SKILL.md
  ~/.claude/skills/my-skill/SKILL.md # 或
  ~/.agents/skills/my-skill/SKILL.md # 或

- 方式三：使用 /skills add（个人级，所有项目生效）
  在 Copilot CLI 中执行：
  /skills add ./skills/my-skill
  该命令会将 Skill 的绝对路径写入全局配置 ~/.copilot/config.json：
  {
  "skill_directories": ["/path/to/your/skills/my-skill"]
  }
  Skill 文件留在原位，Copilot 只是记录了路径引用（类似"书签"）。在原位修改 Skill 内容后，Copilot 下次加载时会立即生效，无需重新 add。
  注意： /skills add 目前只支持添加为个人级 Skill（写入全局配置），无法限定为仅当前项目生效。如需项目级 Skill，请使用方式一。
